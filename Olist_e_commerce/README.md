## Pet-project Анализ датасета e-commerce платформы Olist.
### **Стек**:
- Python (Pandas, NumPy, matplotlib, Seaborn, Scipy), Jupyter
- PowerBI
- PostgreSQL

Для работы jupyter_notebook и SQL-скриптов, требуется импорт с датасета с kaggle,
ниже описано в каком виде должны быть данные для корректной работы без вмешательства.
### Ожидаемые данные:
- 'olist_customers_dataset.csv'
- 'olist_geolocation_dataset.csv'
- 'olist_order_items_dataset.csv'
- 'olist_order_payments_dataset.csv'
- 'olist_order_reviews_dataset.csv'
- 'olist_orders_dataset.csv'
- 'olist_products_dataset.csv'
- 'olist_sellers_dataset.csv'
- 'product_category_name_translation.csv'

### Содержание папок репозитория:
- **View** содержит .pbix .png файлы, содержащие дашборд.
- **data** содержит ссылку на датасет, к сожалению данные в нужном виде нельзя прикрепить из-за веса.
- **jupyter_notebook** содержит код и html версию с выводами для ознакомления с проведённой работой над датасетом.
- **SQL_scripts** Содержит скрипты для создания каркаса бд, очистки от пропусков и вывода данных для построения дашборда
